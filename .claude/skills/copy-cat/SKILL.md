## 用途
「複製貓」：拿一份既有的 eNews MIP HTML 當**版型範例（template）**，再拿另一份 HTML 當**資料內容（content）**，產出一個**新檔案** —— 保留版型的結構、樣式、排版與品牌元素，但把裡面的文字／資料換成內容檔的新資料。

典型情境：用去年（或某語系）已上線、排版乾淨的 Synology eNews 當版型，套入今年（或某活動）由外部／其他來源提供的新文案，快速生出新一期、版型一致的 eNews。

相關脈絡：eNews 專案的 MIP HTML 在 `\\10.17.6.141\01_workout\eNews\cypress\cypress-framework\html_finalToMIP\`。

## 輸入
使用者會指定兩個檔案（路徑通常在上述網路磁碟）：
- **版型範例檔**：決定最終的 HTML 結構、CSS 樣式、排版、品牌元素（如 Synology logo、頁尾、社群列）。
- **資料內容檔**：提供要填入的新文字／資料（標題、日期、內文段落、條列、優惠碼、連結、圖檔網址等）。

若使用者只給一個檔、或沒講清楚誰是版型誰是內容，先問清楚再動手。

### 建議的輸入格式
請使用者用以下格式下指令最不易出錯（兩檔在同一資料夾時）：

```
檔案路徑：<共用資料夾的完整路徑>

版型檔：<版型的檔名.html>

內容檔：<內容的檔名.html>
```

- 共用資料夾只在 `檔案路徑：` 寫一次，下面兩行只寫檔名，避免把整條路徑重複貼造成 `\\\` 壞路徑。
- 檔名含空格或括號（如 `...CKNL (3).html`）直接照打，不必加引號。
- 兩檔若**不在同一資料夾**，省略 `檔案路徑：` 那行，改在 `版型檔：`／`內容檔：` 各寫完整路徑。

解析參數時的注意事項：
- 若路徑出現重複串接（同一段路徑出現兩次、或 `\\\` 這種多重反斜線）、或路徑指到資料夾而非 `.html` 檔，**先停下來向使用者確認正確檔名**，不要硬猜。
- 可先列出該資料夾內容，協助使用者挑出正確的版型檔／內容檔。

## 執行流程

1. **讀兩個檔**：用 Read 完整讀取版型檔與內容檔，分別理解兩者的「區塊結構」。

2. **建立區塊對照**：把版型檔拆成有意義的內容區塊（表頭 / 引導句 / banner / 內文 / 條列 / 主題 / CTA 按鈕 / 出展或合作方資訊 / 頁尾），逐一對應到內容檔裡相對應的新資料。沿用版型檔的 table 結構、行高、style，**只換文字與必要的圖檔網址**，不重排版型。

3. **先問三個關鍵決策**（用 AskUserQuestion，除非使用者已說明）：
   - **佔位／提示文字怎麼處理**：內容檔常有「待填」標記（如粉紅色 `[...]`、`[顧客名]`、「請自訂」提醒）。選項：(a) 沿用版型檔原本對應的既有內容；(b) 原樣保留佔位提示當草稿；(c) 移除佔位、只留通用文字。
   - **圖檔用哪一版**：版型檔的舊圖 vs 內容檔的新圖；逐類（logo、banner、icon）判斷。
   - **輸出檔名與位置**：預設與來源同資料夾，檔名沿用慣例或使用者指定。

4. **產出新檔**：用 Write 另存成**新檔案**（除非使用者要求覆蓋），不要改動版型檔或內容檔本身。

5. **處理衝突要明講**：當某區塊在內容檔沒有對應素材（例如版型有聯名 banner 大圖、但內容檔沒有該圖），不要硬塞會造成錯誤的舊素材（如把去年的年份大圖放進今年的信）。改用內容檔本身合理的做法重建，並在回報時**明確標示這個決策**，請使用者確認或補素材。

6. **回報**：列出「版型 → 資料」的區塊對照表（哪塊來自版型、哪塊換成新資料、混合的標明），並點出需要使用者留意的決策點。提醒檔案位於網路磁碟，與本 repo 版控不同；建議實際寄測郵件確認排版（尤其 Outlook）。

## 慣例
- 版型的品牌元素（logo、頁尾、社群列、退訂列）預設沿用版型檔，因為那代表發信方。
- 年份、日期、主題、優惠碼、長段文案這類**年度／活動專屬資料**，換成內容檔的新值。
- 圖檔若改用內容檔的新圖，注意背景色相容性（例如白色 logo 要配深底）；背景圖相關問題可搭配 background-image-fix skill 處理。
